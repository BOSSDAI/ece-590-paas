From C++ to Java

Method Dispatch

  • Design principles addressed

    • Least Surprise: When we write a child class, we typically want to override the correspending method. If we forget to use the keyword "virtual" in C++, there will be some weird bugs
    • Open / Close: If we want to change some methods of a child class from static dispatch to dynamic dispatch, we must change its parent class, adding "virtual". However, if there are some other child classes of this parent class use static dispatch, there will be bugs.

Memory allocation for objects

  • Design principles addressed

    • Least surprise: no strange bugs from freerelated errors.
    • Better abstraction: don’t need to know where memory is allocated or when to free.
    • Single responsibility: we desgin a method and it don't need to avoid leak memory.
